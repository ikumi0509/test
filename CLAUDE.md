# プロジェクトルール

## 学習メモシステム

このリポジトリには学習メモの管理・可視化システムがあります。

### メモ作成（自然な会話で対応）

ユーザーが「〇〇のメモを作って」「〇〇を学んだ」「〇〇をやりたい」などメモ作成を依頼した場合、以下の手順で対応してください：

1. **会話内容から以下を判定する**:
   - `title`: メモのタイトル（簡潔に）
   - `content`: 詳細な内容
   - `type`: `learned`（学んだこと）/ `todo`（やりたいこと）/ `reflection`（振り返り・感想）
   - `tags`: 関連するタグ（技術名、カテゴリなど）

2. **Pythonスクリプトでメモを保存する**:
   ```bash
   python3 memo_data.py add --title "タイトル" --content "内容" --type learned --tags tag1 tag2
   ```

3. **GitHubにプッシュする**:
   ```bash
   git add memos.json
   git commit -m "メモ追加: タイトル"
   git push
   ```

4. **ユーザーに保存完了を報告する**（タイトル、タイプ、タグを表示）

### 可視化（手動指示時のみ）

ユーザーが「グラフを更新して」「可視化して」などと言った場合のみ：

```bash
python3 memo_visualize.py
git add memos_graph.html
git commit -m "メモグラフを更新"
git push
```

### メモのタイプ判定ガイド

| ユーザーの表現 | type |
|---|---|
| 「〇〇を学んだ」「〇〇がわかった」「〇〇を知った」 | learned |
| 「〇〇をやりたい」「〇〇を実装したい」「〇〇を勉強したい」 | todo |
| 「今日の振り返り」「〇〇だと思った」「〇〇が楽しかった」 | reflection |
